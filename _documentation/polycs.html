Compressive sensing¶

Finding coefficients via compressive sensing.

class equadratures.polycs.Polycs(parameters, basis, training_inputs=None, sampling=None, no_of_points=None, fun=None, training_outputs=None, quadrature_rule=None, no_of_quad_points=None)[source]¶

This class defines a Polycs (polynomial via compressive sensing) object.

Parameters

  • training_inputs (array) – Points at which the model is evaluated.

  • basis (IndexSet) – An instance of the IndexSet class, in case the user wants to overwrite the indices that are obtained using the orders of the univariate parameters in Parameters uq_parameters. The latter corresponds to a tensor grid index set and is the default option if no basis parameter input is given.

  • parameters (list) – List of instances of Parameters class.

  • training_outputs (array) – Column vector of regression targets corresponding to each row of training_inputs. Either this or fun should be specified, but not both. If this is specified, training_inputs must also be specified.

  • fun (callable) – Function to evaluate training_inputs on to obtain regression targets automatically. Either this or fun should be specified, but not both.

  • sampling (str) – Sampling method for non-data-driven model. Valid sampling methods include “standard”, “asymptotic” and “dlm”.

  • quadrature_rule (str) – Quadrature rule. Can be ‘qmc’ or ‘tensor grid’. Refer to documentation for poly.py for more details.

  • no_of_quad_points (int) – Number of quadrature points in case qmc is chosen for quadrature rule. Default number is 10000. Recommended to set to a low number (e.g. 1) when quadrature points are not needed (when statistics need not be computed, for instance).
