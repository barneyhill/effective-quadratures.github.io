Parameter¶

A parameter—i.e., the definition of a univariate variable—is one the main building blocks in Effective Quadratures. It characterized by its distribution (and support).

Definition of a univariate parameter.

class equadratures.parameter.Parameter(order, distribution, endpoints=False, shape_parameter_A=None, shape_parameter_B=None, lower=None, upper=None, data=None)[source]¶

This class defines a univariate parameter. Below are details of its constructor.

Parameters

  • lower (float) – Lower bound for the parameter.

  • upper (float) – Upper bound for the parameter.

  • order (int) – Order of the parameter.

  • param_type (str) – The type of distribution that characterizes the parameter. Options include chebyshev (arcsine), gaussian, truncated-gaussian, beta, cauchy, exponential, uniform, gamma, weibull. If no string is provided, a uniform distribution is assumed. If the user provides data, and would like to generate orthogonal polynomials (and quadrature rules) based on the data, they can set this option to be custom.

  • shape_parameter_A (float) – Most of the aforementioned distributions are characterized by two shape parameters. For instance, in the case of a gaussian (or truncated-gaussian), this represents the mean. In the case of a beta distribution this represents the alpha value. For a uniform distribution this input is not required.

  • shape_parameter_B (float) – This is the second shape parameter that characterizes the distribution selected. In the case of a gaussian or truncated-gaussian, this is the variance.

  • data (numpy.ndarray) – A data-set with shape (number_of_data_points, 2), where the first column comprises of parameter values, while the second column corresponds to the data observations. This input should only be used with the custom distribution.

  • endpoints (bool) – If set to True, then the quadrature points and weights will have end-points, based on Gauss-Lobatto quadrature rules.
